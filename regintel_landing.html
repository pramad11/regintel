Multi-Agent Architecture

Eight specialist agents.
One unified intelligence.

RegIntel orchestrates a pipeline of domain-expert AI agents, each mastering a distinct layer of the PV regulatory stack — from AE intake through E2B R3 XML generation.

Agent 01
Intake Agent
Structured elicitation of product identity, development phase, AE details, Day 0, and sponsor context through guided dialogue.
→ classifier
Agent 02
Classifier Agent
MedDRA PT/SOC coding of verbatim AE terms. ICH E2A seriousness assessment. SUSAR determination for CT cases.
→ obligation
Agent 03
Geography Agent
Maps trial countries to responsible HAs. Applies domestic vs. foreign reporter logic per jurisdiction. Flags local language requirements.
→ expectedness
Agent 04
Expectedness Agent
Compares MedDRA PT against uploaded IB, SmPC, or USPI. Determines listed vs. unlisted status. Confirms SUSAR trigger.
→ obligation
Agent 05
Obligation Agent
Queries the Regulatory Obligation Database across all selected HAs. Resolves expedited, periodic, and aggregate reporting requirements with full citation.
→ timeline
Agent 06
Timeline Agent
Calculates precise due dates from Day 0. Applies holiday calendars per HA. Generates follow-up report schedules and periodic report calendars.
→ narrative
Agent 07
Narrative Agent
Drafts ICH E2B R3-compliant case narratives (H.1/H.3/H.4). Generates CIOMS I text. Flags translation requirements for ANVISA, PMDA, NMPA.
→ output
Agent 08
Output Agent
Assembles the full obligation matrix. Generates E2B R3 XML. Produces submission calendar, gap analysis, and HA-specific cover letters.
→ export

Technical Architecture

Regulatory Obligation Database.
Rules engine built for PV.

The ROD is a structured PostgreSQL rules engine normalizing reporting obligations from 187 HAs into a queryable, always-current dataset.

reporting_obligation
get_obligations()
ha_feed_log
CREATE TABLE reporting_obligation (
  obligation_id    UUID         -- primary key
  ha_id            UUID         -- → health_authority
  product_type     product_type_enum[]
  phase            development_phase_enum[]
  seriousness      seriousness_criteria_enum[]
  expectedness     expectedness_enum[]
  domestic_foreign domestic_foreign_enum
  report_name      VARCHAR(200)  -- 'SUSAR', 'MedWatch 3500A'
  report_type      VARCHAR(50)   -- '7-day', '15-day', 'DSUR'
  timeline_days    INTEGER       -- NULL for periodic
  timeline_basis   timeline_basis_enum
  report_format    report_format_enum[]
  submission_route submission_route_enum[]
  is_periodic      BOOLEAN
  periodic_report_ref VARCHAR(100) -- 'DSUR', 'PSUR', 'PADER'
  conditions_text  TEXT
  active           BOOLEAN       -- live / sunset toggle
  last_updated     TIMESTAMPTZ   -- auto-updated by HA scraper
);
